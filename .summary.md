## Objective
- Place a video demo in the hero section, remove the separate demo section, clean up project cards (duplicate settings + invite dialog), fix middleware routing, implement a complete realtime notification system, and begin merging Invitations + Access Requests into a unified Access Center

## Important Details
- **Stale build cache** (`vendor-chunks/lucide-react.js` missing) is fixed by deleting `.next` folder; this happens when Next.js incremental cache holds references to deleted chunk files
- **Middleware** now protects: `/dashboard(.*)`, `/project(.*)`, `/editor(.*)`, `/sync`, `/invitations(.*)`, `/profile(.*)`, `/settings(.*)`, `/requests(.*)`, `/access(.*)`, `/notifications(.*)`; `/invitations/[token]` is public; authenticated users only redirect from `/` — not from public invitation links; `/invitations` and `/requests` redirect to `/access`
- **Notification model** in Prisma has: `actorId` (nullable), `metadata` (Json), indexes on `[userId, createdAt]` and `[userId, type]`, plus `actor` relation linked to `User.actedNotifications`
- **Notification system** uses existing `AppGateway.notifyUser()` for realtime delivery; each notification.create emits both `notification` and `notification-unread-count` socket events
- **Existing components overwritten**: `components/NotificationBell.tsx` (editor header) and `components/notifications/NotificationBell.tsx` (sidebar) both exist; the editor one now uses the new options-based hook
- **Prisma schema drift** was resolved with `prisma db push --accept-data-loss` (DB had hand-rolled columns not in migration history)
- **Backend `AccessModule`** reuses `InvitationsService` and `AccessRequestsService` via exports — `AccessRequestsModule` was fixed to export its service and use `PrismaModule`
